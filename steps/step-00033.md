# Step 00033 — Create .env.example

## Context

Documents required environment variables for local Docker Compose. The actual `.env` file is gitignored.

## Action

Create the file `.env.example` at the repo root with the following exact content:

```
# MySQL root password (used by the mysql Docker service)
MYSQL_ROOT_PASSWORD=changeme

# Application DB password (used by pooled-app and unpooled-app)
DB_PASSWORD=changeme
```

## Expected result

`.env.example` exists at the repo root. Developers copy it to `.env` and fill in real values before running `docker compose up`.
