# Step 00045 — Create k8s/unpooled-app.yaml

## Context
Kubernetes Deployment and Service for the unpooled NestJS app. Connects directly to the unpooled Aurora cluster endpoint via VPC peering. Uses `DB_CONNECTION_MODE=per-request`.

## Action
Create the file `k8s/unpooled-app.yaml` with the following exact content:

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: unpooled-app
  namespace: pooling-poc
  labels:
    app: unpooled-app
spec:
  replicas: 1
  selector:
    matchLabels:
      app: unpooled-app
  template:
    metadata:
      labels:
        app: unpooled-app
    spec:
      containers:
        - name: unpooled-app
          image: ${ECR_REPO_UNPOOLED}:${IMAGE_TAG}
          imagePullPolicy: IfNotPresent
          ports:
            - containerPort: 3000
          env:
            - name: DB_HOST
              value: "${UNPOOLED_DB_HOST}"
            - name: DB_PORT
              value: "3306"
            - name: DB_USER
              value: "admin"
            - name: DB_PASSWORD
              valueFrom:
                secretKeyRef:
                  name: unpooled-db-secret
                  key: password
            - name: DB_NAME
              value: "pooling_poc"
            - name: DB_SSL
              value: "true"
            - name: DB_CONNECTION_MODE
              value: "per-request"
          readinessProbe:
            httpGet:
              path: /health
              port: 3000
            initialDelaySeconds: 30
            periodSeconds: 600
          livenessProbe:
            httpGet:
              path: /health
              port: 3000
            initialDelaySeconds: 30
            periodSeconds: 60
---
apiVersion: v1
kind: Service
metadata:
  name: unpooled-app
  namespace: pooling-poc
spec:
  selector:
    app: unpooled-app
  ports:
    - port: 80
      targetPort: 3000
```

## Expected result
`k8s/unpooled-app.yaml` exists.
