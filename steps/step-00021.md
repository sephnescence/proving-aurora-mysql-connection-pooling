# Step ✅ 00021 — Scaffold @pooling-poc/unpooled-app with NestJS CLI

## Context

Create the `apps/unpooled-app` NestJS application using the NestJS CLI. Identical structure to pooled-app; differences are environment variables only.

## Pre-requisite

NestJS CLI is available as a root dev dependency (installed in step-00014).

## Commands

Run from the repo root:

```bash
pnpm exec nest new apps/unpooled-app --package-manager pnpm --skip-git --strict
```

Then update `apps/unpooled-app/package.json`:

- Change `"name"` to `"@pooling-poc/unpooled-app"`
- Add `"private": true`

Delete the generated placeholder files:

```bash
rm apps/unpooled-app/src/app.controller.spec.ts
rm apps/unpooled-app/src/app.controller.ts
rm apps/unpooled-app/src/app.service.ts
```

Clean up the references to the deleted files in `apps/unpooled-app/src/app.module.ts`:

```typescript
import { Module } from '@nestjs/common'

@Module({
  imports: [],
})
export class AppModule {}
```

Fix the generated `apps/unpooled-app/tsconfig.json` — the CLI generates `"module": "nodenext"` which requires explicit `.js` extensions on relative imports and breaks NestJS code. Replace with:

```json
{
  "compilerOptions": {
    "module": "commonjs",
    "declaration": true,
    "removeComments": true,
    "emitDecoratorMetadata": true,
    "experimentalDecorators": true,
    "allowSyntheticDefaultImports": true,
    "target": "ES2023",
    "sourceMap": true,
    "outDir": "./dist",
    "baseUrl": "./",
    "incremental": true,
    "skipLibCheck": true,
    "strictNullChecks": true,
    "noImplicitAny": true,
    "strictBindCallApply": true,
    "forceConsistentCasingInFileNames": true,
    "noFallthroughCasesInSwitch": true,
    "esModuleInterop": true,
    "isolatedModules": true
  }
}
```

Delete the generated Jest and ESLint files, and replace Jest with Vitest (same as step-00015b):

```bash
rm -rf apps/unpooled-app/test
rm apps/unpooled-app/eslint.config.mjs
rm apps/unpooled-app/.prettierrc
```

Replace the `scripts`, `devDependencies`, and remove the `jest` config block from `apps/unpooled-app/package.json` — mirror the final state of `apps/pooled-app/package.json` but with the name `@pooling-poc/unpooled-app`.

Create `apps/unpooled-app/vitest.config.ts`:

```typescript
import { defineConfig } from 'vitest/config'

export default defineConfig({
  test: {
    globals: true,
    environment: 'node',
  },
})
```

Run `pnpm install` from the repo root to update the lockfile.

## Expected result

`apps/unpooled-app/` exists with a valid NestJS project skeleton. `apps/unpooled-app/package.json` has `"name": "@pooling-poc/unpooled-app"`. No Jest or ESLint dependencies. `tsconfig.json` uses `"module": "commonjs"`.
