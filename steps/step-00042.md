# Step 00042 — Create k8s/unpooled-app-external-secret.yaml

## Context

External Secrets Operator pulls the DB password for the unpooled Aurora cluster from Secrets Manager and creates `unpooled-db-secret`. Shares the same `eso-service-account` as the pooled secret store. The `${UNPOOLED_DB_SECRET_ARN}` placeholder is replaced by `envsubst` during deployment.

## Action

Create the file `k8s/unpooled-app-external-secret.yaml` with the following exact content:

```yaml
apiVersion: external-secrets.io/v1beta1
kind: SecretStore
metadata:
  name: unpooled-secret-store
  namespace: pooling-poc
spec:
  provider:
    aws:
      service: SecretsManager
      region: ap-southeast-2
      auth:
        jwt:
          serviceAccountRef:
            name: eso-service-account
---
apiVersion: external-secrets.io/v1beta1
kind: ExternalSecret
metadata:
  name: unpooled-db-external-secret
  namespace: pooling-poc
spec:
  refreshInterval: 1h
  secretStoreRef:
    name: unpooled-secret-store
    kind: SecretStore
  target:
    name: unpooled-db-secret
    creationPolicy: Owner
  data:
    - secretKey: password
      remoteRef:
        key: ${UNPOOLED_DB_SECRET_ARN}
        property: password
```

## Expected result

`k8s/unpooled-app-external-secret.yaml` exists.
