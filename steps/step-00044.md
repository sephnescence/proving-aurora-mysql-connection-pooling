# Step 00044 — Create k8s/pooled-app.yaml

## Context

Kubernetes Deployment and Service for the pooled NestJS app. Connects to the RDS Proxy endpoint. Uses `DB_CONNECTION_MODE=persistent`. The `${ECR_REPO_POOLED}` and `${IMAGE_TAG}` placeholders are substituted by `envsubst` during deployment.

## Action

Create the file `k8s/pooled-app.yaml` with the following exact content:

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: pooled-app
  namespace: pooling-poc
  labels:
    app: pooled-app
spec:
  replicas: 1
  selector:
    matchLabels:
      app: pooled-app
  template:
    metadata:
      labels:
        app: pooled-app
    spec:
      containers:
        - name: pooled-app
          image: ${ECR_REPO_POOLED}:${IMAGE_TAG}
          imagePullPolicy: IfNotPresent
          ports:
            - containerPort: 3000
          env:
            - name: DB_HOST
              value: '${POOLED_DB_HOST}'
            - name: DB_PORT
              value: '3306'
            - name: DB_USER
              value: 'admin'
            - name: DB_PASSWORD
              valueFrom:
                secretKeyRef:
                  name: pooled-db-secret
                  key: password
            - name: DB_NAME
              value: 'pooling_poc'
            - name: DB_SSL
              value: 'true'
            - name: DB_CONNECTION_MODE
              value: 'persistent'
          readinessProbe:
            httpGet:
              path: /health
              port: 3000
            initialDelaySeconds: 30
            periodSeconds: 600
          livenessProbe:
            httpGet:
              path: /health
              port: 3000
            initialDelaySeconds: 30
            periodSeconds: 60
---
apiVersion: v1
kind: Service
metadata:
  name: pooled-app
  namespace: pooling-poc
spec:
  selector:
    app: pooled-app
  ports:
    - port: 80
      targetPort: 3000
```

## Expected result

`k8s/pooled-app.yaml` exists.
