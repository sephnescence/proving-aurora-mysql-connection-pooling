# Step 00046 — Create k8s/ingress.yaml

## Context
AWS Application Load Balancer (internet-facing) created by the AWS Load Balancer Controller. Path-based routing: `/pooled/*` → pooled-app (prefix stripped), `/unpooled/*` → unpooled-app (prefix stripped).

## Action
Create the file `k8s/ingress.yaml` with the following exact content:

```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: pooling-poc-ingress
  namespace: pooling-poc
  annotations:
    kubernetes.io/ingress.class: alb
    alb.ingress.kubernetes.io/scheme: internet-facing
    alb.ingress.kubernetes.io/target-type: ip
    alb.ingress.kubernetes.io/listen-ports: '[{"HTTP": 80}]'
spec:
  rules:
    - http:
        paths:
          - path: /pooled
            pathType: Prefix
            backend:
              service:
                name: pooled-app
                port:
                  number: 80
          - path: /unpooled
            pathType: Prefix
            backend:
              service:
                name: unpooled-app
                port:
                  number: 80
```

## Note on prefix stripping
The ALB forwards requests stripping the `/pooled` and `/unpooled` prefixes so the NestJS apps receive `/processlist` and `/health` directly. This is achieved with the `alb.ingress.kubernetes.io/actions` pattern or by configuring target group rules. Append the following annotation to the ingress metadata if prefix-stripping actions are required by your ALB Controller version:

```yaml
    alb.ingress.kubernetes.io/actions.pooled-app: >
      {"type":"forward","forwardConfig":{"targetGroups":[{"serviceName":"pooled-app","servicePort":80}]}}
```

Verify behaviour with your installed ALB Controller version after deployment.

## Expected result
`k8s/ingress.yaml` exists.
