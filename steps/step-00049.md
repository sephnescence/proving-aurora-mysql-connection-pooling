# Step ✅ 00049 — Append to infra/main.yml — IAM IRSA Roles, CloudWatch, and Outputs

## Context

Final section of `infra/main.yml`. Appends the four IAM roles required for IRSA/GitHub Actions (ESO, ALB Controller, Fluent Bit, Artillery), the two CloudWatch log groups, and the CloudWatch dashboard. Also adds the Outputs section used by deploy and artillery workflows.

## Action

Append the following content to `infra/main.yml` (after the EKS node group resource):

```yaml

  # ── IRSA: ESO ────────────────────────────────────────────────────────────────

  EsoIrsaRole:
    Type: AWS::IAM::Role
    Properties:
      RoleName: aurora-pooling-poc-eso-irsa-role
      AssumeRolePolicyDocument:
        Version: "2012-10-17"
        Statement:
          - Effect: Allow
            Principal:
              Federated: !Sub "arn:aws:iam::${AWS::AccountId}:oidc-provider/${EksCluster.OpenIdConnectIssuerUrl}"
            Action: sts:AssumeRoleWithWebIdentity
            Condition:
              StringEquals:
                "${EksCluster.OpenIdConnectIssuerUrl}:aud": sts.amazonaws.com
                "${EksCluster.OpenIdConnectIssuerUrl}:sub": "system:serviceaccount:pooling-poc:eso-service-account"
      Policies:
        - PolicyName: SecretsManagerRead
          PolicyDocument:
            Version: "2012-10-17"
            Statement:
              - Effect: Allow
                Action:
                  - secretsmanager:GetSecretValue
                  - secretsmanager:DescribeSecret
                Resource:
                  - !GetAtt PooledAuroraCluster.MasterUserSecret.SecretArn
                  - !GetAtt UnpooledAuroraCluster.MasterUserSecret.SecretArn

  # ── IRSA: ALB Controller ─────────────────────────────────────────────────────

  AlbControllerIrsaRole:
    Type: AWS::IAM::Role
    Properties:
      RoleName: aurora-pooling-poc-alb-controller-irsa-role
      AssumeRolePolicyDocument:
        Version: "2012-10-17"
        Statement:
          - Effect: Allow
            Principal:
              Federated: !Sub "arn:aws:iam::${AWS::AccountId}:oidc-provider/${EksCluster.OpenIdConnectIssuerUrl}"
            Action: sts:AssumeRoleWithWebIdentity
            Condition:
              StringEquals:
                "${EksCluster.OpenIdConnectIssuerUrl}:aud": sts.amazonaws.com
                "${EksCluster.OpenIdConnectIssuerUrl}:sub": "system:serviceaccount:kube-system:aws-load-balancer-controller"
      ManagedPolicyArns:
        - !Sub "arn:aws:iam::${AWS::AccountId}:policy/AWSLoadBalancerControllerIAMPolicy"

  # ── IRSA: Fluent Bit ─────────────────────────────────────────────────────────

  FluentBitIrsaRole:
    Type: AWS::IAM::Role
    Properties:
      RoleName: aurora-pooling-poc-fluent-bit-irsa-role
      AssumeRolePolicyDocument:
        Version: "2012-10-17"
        Statement:
          - Effect: Allow
            Principal:
              Federated: !Sub "arn:aws:iam::${AWS::AccountId}:oidc-provider/${EksCluster.OpenIdConnectIssuerUrl}"
            Action: sts:AssumeRoleWithWebIdentity
            Condition:
              StringEquals:
                "${EksCluster.OpenIdConnectIssuerUrl}:aud": sts.amazonaws.com
                "${EksCluster.OpenIdConnectIssuerUrl}:sub": "system:serviceaccount:amazon-cloudwatch:fluent-bit"
      Policies:
        - PolicyName: CloudWatchLogsWrite
          PolicyDocument:
            Version: "2012-10-17"
            Statement:
              - Effect: Allow
                Action:
                  - logs:CreateLogGroup
                  - logs:CreateLogStream
                  - logs:PutLogEvents
                  - logs:DescribeLogStreams
                Resource: "*"

  # ── IAM: Artillery Role (GitHub OIDC) ────────────────────────────────────────

  ArtilleryRole:
    Type: AWS::IAM::Role
    Properties:
      RoleName: aurora-pooling-poc-artillery-role
      AssumeRolePolicyDocument:
        Version: "2012-10-17"
        Statement:
          - Effect: Allow
            Principal:
              Federated:
                Fn::ImportValue: aurora-pooling-poc-bootstrap-DeployRoleArn
            Action: sts:AssumeRole
            Condition:
              StringLike:
                token.actions.githubusercontent.com:sub: "repo:*:ref:refs/heads/main"
      Policies:
        - PolicyName: ArtilleryPermissions
          PolicyDocument:
            Version: "2012-10-17"
            Statement:
              - Effect: Allow
                Action:
                  - cloudformation:DescribeStacks
                Resource: !Sub "arn:aws:cloudformation:${AWS::Region}:${AWS::AccountId}:stack/aurora-pooling-poc/*"
              - Effect: Allow
                Action:
                  - logs:StartQuery
                  - logs:GetQueryResults
                  - logs:DescribeLogGroups
                Resource: "*"

  # ── CloudWatch Log Groups ────────────────────────────────────────────────────

  PooledLogGroup:
    Type: AWS::Logs::LogGroup
    Properties:
      LogGroupName: /aurora-pooling-poc/pooled-app
      RetentionInDays: 7

  UnpooledLogGroup:
    Type: AWS::Logs::LogGroup
    Properties:
      LogGroupName: /aurora-pooling-poc/unpooled-app
      RetentionInDays: 7

  # ── CloudWatch Dashboard ─────────────────────────────────────────────────────

    Type: AWS::CloudWatch::Dashboard
    Properties:
      DashboardName: aurora-pooling-poc
      DashboardBody: !Sub |
        {
          "widgets": [
            {

                "region": "${AWS::Region}",

            {
              "type": "log",

                "title": "Unpooled — sequential",
                "query": "SOURCE '/aurora-pooling-poc/unpooled-app' | fields @timestamp, testType, rowCount, rows | filter testType = \"sequential\" | sort @timestamp desc | limit 50",
                "region": "${AWS::Region}",

                "period": 3600
              }

Outputs:
```

## Note on AlbControllerIrsaRole

The `AWSLoadBalancerControllerIAMPolicy` referenced here must exist in your account before the main stack deploys. The command to create it is covered in **step 57** (`DEPLOYMENT.md`).

## Note on ArtilleryRole trust policy

The ArtilleryRole trust above uses the deploy role ARN as principal. Update the trust to use the GitHub OIDC provider directly (same pattern as DeployRole in bootstrap.yml), referencing the existing OIDC provider via `arn:aws:iam::${AWS::AccountId}:oidc-provider/token.actions.githubusercontent.com`.

## Expected result

`infra/main.yml` is complete with all resources and an Outputs section.
