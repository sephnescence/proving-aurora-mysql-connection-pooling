# Step 00051b — Bootstrap and deploy infrastructure manually

## Context

Before the deploy workflow (step 51) can run via GitHub Actions, two one-time AWS account prerequisites must be in place. Once those are done, you can either trigger the deploy workflow from GitHub or run the equivalent AWS CLI commands directly from your terminal to get the infrastructure up so you can continue with step 51c.

## Action

### 1. Deploy the bootstrap stack (one-time)

```bash
aws cloudformation deploy \
  --template-file infra/bootstrap.yml \
  --stack-name aurora-pooling-poc-bootstrap \
  --parameter-overrides GitHubOrg=<your-github-org> \
  --capabilities CAPABILITY_NAMED_IAM \
  --region ap-southeast-2
```

### 2. Create the ALB Controller IAM policy (one-time)

```bash
curl -o alb-iam-policy.json \
  https://raw.githubusercontent.com/kubernetes-sigs/aws-load-balancer-controller/main/docs/install/iam_policy.json

aws iam create-policy \
  --policy-name AWSLoadBalancerControllerIAMPolicy \
  --policy-document file://alb-iam-policy.json \
  --region ap-southeast-2
```

### 3. Configure the GitHub repository secret

In your GitHub repository go to **Settings → Secrets and variables → Actions** and add:

| Secret | Value |
|--------|-------|
| `AWS_ACCOUNT_ID` | Your 12-digit AWS account number |

### 4. Deploy the main stack

Either trigger `.github/workflows/deploy.yml` manually from the GitHub Actions tab, or run the equivalent steps directly from your terminal:

```bash
# Get ECR URIs
ECR_REPO_POOLED=$(aws cloudformation describe-stacks \
  --stack-name aurora-pooling-poc-bootstrap \
  --query "Stacks[0].Outputs[?OutputKey=='ECRPooledAppUri'].OutputValue" \
  --output text)
ECR_REPO_UNPOOLED=$(aws cloudformation describe-stacks \
  --stack-name aurora-pooling-poc-bootstrap \
  --query "Stacks[0].Outputs[?OutputKey=='ECRUnpooledAppUri'].OutputValue" \
  --output text)

# Build and push images
IMAGE_TAG=$(git rev-parse HEAD)
aws ecr get-login-password --region ap-southeast-2 | \
  docker login --username AWS --password-stdin $ECR_REPO_POOLED
docker build -f apps/pooled-app/Dockerfile -t $ECR_REPO_POOLED:$IMAGE_TAG .
docker push $ECR_REPO_POOLED:$IMAGE_TAG
docker build -f apps/unpooled-app/Dockerfile -t $ECR_REPO_UNPOOLED:$IMAGE_TAG .
docker push $ECR_REPO_UNPOOLED:$IMAGE_TAG

# Deploy main stack
aws cloudformation deploy \
  --template-file infra/main.yml \
  --stack-name aurora-pooling-poc \
  --capabilities CAPABILITY_NAMED_IAM \
  --parameter-overrides \
    ImageTagPooled=$IMAGE_TAG \
    ImageTagUnpooled=$IMAGE_TAG \
  --region ap-southeast-2

# Update kubeconfig
aws eks update-kubeconfig \
  --name aurora-pooling-poc-eks \
  --region ap-southeast-2
```

### 5. Apply Kubernetes manifests

```bash
# Get stack outputs
POOLED_DB_HOST=$(aws cloudformation describe-stacks --stack-name aurora-pooling-poc \
  --query "Stacks[0].Outputs[?OutputKey=='PooledDbHost'].OutputValue" --output text)
UNPOOLED_DB_HOST=$(aws cloudformation describe-stacks --stack-name aurora-pooling-poc \
  --query "Stacks[0].Outputs[?OutputKey=='UnpooledDbHost'].OutputValue" --output text)
POOLED_DB_SECRET_ARN=$(aws cloudformation describe-stacks --stack-name aurora-pooling-poc \
  --query "Stacks[0].Outputs[?OutputKey=='PooledDbSecretArn'].OutputValue" --output text)
UNPOOLED_DB_SECRET_ARN=$(aws cloudformation describe-stacks --stack-name aurora-pooling-poc \
  --query "Stacks[0].Outputs[?OutputKey=='UnpooledDbSecretArn'].OutputValue" --output text)
ESO_ROLE_ARN=$(aws cloudformation describe-stacks --stack-name aurora-pooling-poc \
  --query "Stacks[0].Outputs[?OutputKey=='EsoRoleArn'].OutputValue" --output text)
ALB_CONTROLLER_ROLE_ARN=$(aws cloudformation describe-stacks --stack-name aurora-pooling-poc \
  --query "Stacks[0].Outputs[?OutputKey=='AlbControllerRoleArn'].OutputValue" --output text)
FLUENT_BIT_ROLE_ARN=$(aws cloudformation describe-stacks --stack-name aurora-pooling-poc \
  --query "Stacks[0].Outputs[?OutputKey=='FluentBitRoleArn'].OutputValue" --output text)

# Install Helm charts
helm repo add external-secrets https://charts.external-secrets.io
helm repo add eks https://aws.github.io/eks-charts
helm repo add aws https://aws.github.io/eks-charts
helm repo update

helm upgrade --install external-secrets external-secrets/external-secrets \
  --namespace external-secrets-system --create-namespace --wait

helm upgrade --install aws-load-balancer-controller eks/aws-load-balancer-controller \
  --namespace kube-system \
  --set clusterName=aurora-pooling-poc-eks \
  --set serviceAccount.annotations."eks\.amazonaws\.com/role-arn"=$ALB_CONTROLLER_ROLE_ARN \
  --wait

helm upgrade --install aws-for-fluent-bit aws/aws-for-fluent-bit \
  --namespace amazon-cloudwatch --create-namespace \
  --set serviceAccount.annotations."eks\.amazonaws\.com/role-arn"=$FLUENT_BIT_ROLE_ARN \
  --set cloudWatch.enabled=true \
  --set cloudWatch.region=ap-southeast-2 \
  --set cloudWatch.logGroupName=/aurora-pooling-poc/$(POD_LABEL_APP) \
  --wait

# Apply manifests
export ECR_REPO_POOLED ECR_REPO_UNPOOLED IMAGE_TAG \
  POOLED_DB_HOST UNPOOLED_DB_HOST \
  POOLED_DB_SECRET_ARN UNPOOLED_DB_SECRET_ARN ESO_ROLE_ARN

for manifest in \
  k8s/namespace.yaml \
  k8s/pooled-app-external-secret.yaml \
  k8s/unpooled-app-external-secret.yaml \
  k8s/pooled-app.yaml \
  k8s/unpooled-app.yaml \
  k8s/ingress.yaml; do
  envsubst < $manifest | kubectl apply -f -
done

kubectl rollout status deployment/pooled-app -n pooling-poc --timeout=300s
kubectl rollout status deployment/unpooled-app -n pooling-poc --timeout=300s
```

## Expected result

EKS is running, both deployments are rolled out, and `kubectl get pods -n pooling-poc` shows both pods as `Running`.
