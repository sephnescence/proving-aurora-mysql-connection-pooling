# Step 00051b — Verify ALB prefix stripping

## Context

The NestJS apps register routes at `/processlist` and `/health` with no path prefix. The ALB ingress routes `/pooled/*` and `/unpooled/*` to each app, so the `/pooled` and `/unpooled` prefixes must be stripped before the request reaches the app.

> **Prerequisites:** EKS must be running, the ALB Controller must be installed, and the ingress must be deployed. All of this happens during step 51 (deploy workflow). Do not attempt this step until that workflow has completed successfully.

## Action

### 1. Test prefix stripping

Retrieve the ALB DNS name:

```bash
kubectl get ingress pooling-poc-ingress -n pooling-poc \
  -o jsonpath='{.status.loadBalancer.ingress[0].hostname}'
```

Then hit both health endpoints:

```bash
curl http://<ALB_DNS>/pooled/health
curl http://<ALB_DNS>/unpooled/health
```

Both should return `200 OK`. If they do, prefix stripping is working — no further action needed.

### 2. If you get 404s — add action annotations

Edit `k8s/ingress.yaml` and add two annotations to the `metadata.annotations` block. The full annotations block should look like this:

```yaml
  annotations:
    kubernetes.io/ingress.class: alb
    alb.ingress.kubernetes.io/scheme: internet-facing
    alb.ingress.kubernetes.io/target-type: ip
    alb.ingress.kubernetes.io/listen-ports: '[{"HTTP": 80}]'
    alb.ingress.kubernetes.io/actions.pooled-app: >
      {"type":"forward","forwardConfig":{"targetGroups":[{"serviceName":"pooled-app","servicePort":80}]}}
    alb.ingress.kubernetes.io/actions.unpooled-app: >
      {"type":"forward","forwardConfig":{"targetGroups":[{"serviceName":"unpooled-app","servicePort":80}]}}
```

Then re-apply:

```bash
kubectl apply -f k8s/ingress.yaml
```

Re-run the `curl` tests from step 1 to confirm both endpoints now return `200 OK`.

## Expected result

`GET /pooled/health` and `GET /unpooled/health` via the ALB both return `200 OK`.
