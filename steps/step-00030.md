```typescript
import { defineConfig } from 'vitest/config'

export default defineConfig({
  test: {
    globals: true,
    environment: 'node',
    testTimeout: 30000,
  },
})
```

## Expected result

`packages/integration-tests/vitest.config.ts` exists.

## Running integration tests

Integration tests make real HTTP requests to both apps. At this stage, the tests expect HTTP 500 because neither app has DB env vars configured — this is intentional. Start both apps in separate terminals:

```bash
# Terminal 1
pnpm --filter @pooling-poc/pooled-app start

# Terminal 2
pnpm --filter @pooling-poc/unpooled-app start
```

The apps will start but return 500 on every request due to missing DB connection config. The tests assert this and should pass. Once Docker Compose is set up (step-00038) and `.env` is configured (step-00039), the tests will be updated to expect 200 (step-00040).
