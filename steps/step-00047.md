# Step 00047 — Create infra/main.yml (CloudFormation) — Part 1: VPCs and Networking

## Context

The main CloudFormation stack is large. It is split across steps 00048–00050 for readability, but all content goes into a single file `infra/main.yml`. This step creates the file with the VPC, subnet, IGW, NAT Gateway, VPC peering, and route table resources.

## Action

Create the file `infra/main.yml` with the following content. Subsequent steps (00049, 00050) will append to this file.

```yaml
AWSTemplateFormatVersion: '2010-09-09'
Description: Main stack — VPCs, EKS, Aurora x2, RDS Proxy, IAM, CloudWatch

Parameters:
  ImageTagPooled:
    Type: String
  ImageTagUnpooled:
    Type: String

Resources:
  # ── VPC A (EKS + Pooled Aurora + RDS Proxy) ─────────────────────────────────

  VpcA:
    Type: AWS::EC2::VPC
    Properties:
      CidrBlock: 10.0.0.0/16
      EnableDnsHostnames: true
      EnableDnsSupport: true
      Tags: [{ Key: Name, Value: aurora-pooling-poc-vpc-a }]

  VpcAPublicSubnetA:
    Type: AWS::EC2::Subnet
    Properties:
      VpcId: !Ref VpcA
      CidrBlock: 10.0.0.0/24
      AvailabilityZone: !Select [0, !GetAZs '']
      MapPublicIpOnLaunch: true
      Tags:
        [
          { Key: Name, Value: vpc-a-public-a },
          { Key: 'kubernetes.io/role/elb', Value: '1' },
        ]

  VpcAPublicSubnetB:
    Type: AWS::EC2::Subnet
    Properties:
      VpcId: !Ref VpcA
      CidrBlock: 10.0.1.0/24
      AvailabilityZone: !Select [1, !GetAZs '']
      MapPublicIpOnLaunch: true
      Tags:
        [
          { Key: Name, Value: vpc-a-public-b },
          { Key: 'kubernetes.io/role/elb', Value: '1' },
        ]

  VpcAPrivateSubnetA:
    Type: AWS::EC2::Subnet
    Properties:
      VpcId: !Ref VpcA
      CidrBlock: 10.0.2.0/24
      AvailabilityZone: !Select [0, !GetAZs '']
      Tags:
        [
          { Key: Name, Value: vpc-a-private-a },
          { Key: 'kubernetes.io/role/internal-elb', Value: '1' },
        ]

  VpcAPrivateSubnetB:
    Type: AWS::EC2::Subnet
    Properties:
      VpcId: !Ref VpcA
      CidrBlock: 10.0.3.0/24
      AvailabilityZone: !Select [1, !GetAZs '']
      Tags:
        [
          { Key: Name, Value: vpc-a-private-b },
          { Key: 'kubernetes.io/role/internal-elb', Value: '1' },
        ]

  IgwA:
    Type: AWS::EC2::InternetGateway
    Properties:
      Tags: [{ Key: Name, Value: aurora-pooling-poc-igw-a }]

  IgwAAttachment:
    Type: AWS::EC2::VPCGatewayAttachment
    Properties:
      VpcId: !Ref VpcA
      InternetGatewayId: !Ref IgwA

  ElasticIpA:
    Type: AWS::EC2::EIP
    Properties:
      Domain: vpc

  NatGatewayA:
    Type: AWS::EC2::NatGateway
    Properties:
      AllocationId: !GetAtt ElasticIpA.AllocationId
      SubnetId: !Ref VpcAPublicSubnetA
      Tags: [{ Key: Name, Value: aurora-pooling-poc-nat-a }]

  PublicRouteTableA:
    Type: AWS::EC2::RouteTable
    Properties:
      VpcId: !Ref VpcA

  PublicRouteA:
    Type: AWS::EC2::Route
    DependsOn: IgwAAttachment
    Properties:
      RouteTableId: !Ref PublicRouteTableA
      DestinationCidrBlock: 0.0.0.0/0
      GatewayId: !Ref IgwA

  PublicSubnetARouteAssoc:
    Type: AWS::EC2::SubnetRouteTableAssociation
    Properties:
      SubnetId: !Ref VpcAPublicSubnetA
      RouteTableId: !Ref PublicRouteTableA

  PublicSubnetBRouteAssoc:
    Type: AWS::EC2::SubnetRouteTableAssociation
    Properties:
      SubnetId: !Ref VpcAPublicSubnetB
      RouteTableId: !Ref PublicRouteTableA

  PrivateRouteTableA:
    Type: AWS::EC2::RouteTable
    Properties:
      VpcId: !Ref VpcA

  PrivateRouteA:
    Type: AWS::EC2::Route
    Properties:
      RouteTableId: !Ref PrivateRouteTableA
      DestinationCidrBlock: 0.0.0.0/0
      NatGatewayId: !Ref NatGatewayA

  PrivateSubnetARouteAssoc:
    Type: AWS::EC2::SubnetRouteTableAssociation
    Properties:
      SubnetId: !Ref VpcAPrivateSubnetA
      RouteTableId: !Ref PrivateRouteTableA

  PrivateSubnetBRouteAssoc:
    Type: AWS::EC2::SubnetRouteTableAssociation
    Properties:
      SubnetId: !Ref VpcAPrivateSubnetB
      RouteTableId: !Ref PrivateRouteTableA

  # ── VPC B (Unpooled Aurora only) ────────────────────────────────────────────

  VpcB:
    Type: AWS::EC2::VPC
    Properties:
      CidrBlock: 10.1.0.0/16
      EnableDnsHostnames: true
      EnableDnsSupport: true
      Tags: [{ Key: Name, Value: aurora-pooling-poc-vpc-b }]

  VpcBPrivateSubnetA:
    Type: AWS::EC2::Subnet
    Properties:
      VpcId: !Ref VpcB
      CidrBlock: 10.1.0.0/24
      AvailabilityZone: !Select [0, !GetAZs '']
      Tags: [{ Key: Name, Value: vpc-b-private-a }]

  VpcBPrivateSubnetB:
    Type: AWS::EC2::Subnet
    Properties:
      VpcId: !Ref VpcB
      CidrBlock: 10.1.1.0/24
      AvailabilityZone: !Select [1, !GetAZs '']
      Tags: [{ Key: Name, Value: vpc-b-private-b }]

  # ── VPC Peering ─────────────────────────────────────────────────────────────

  VpcPeeringAB:
    Type: AWS::EC2::VPCPeeringConnection
    Properties:
      VpcId: !Ref VpcA
      PeerVpcId: !Ref VpcB
      Tags: [{ Key: Name, Value: aurora-pooling-poc-peering-ab }]

  PeeringRouteAtoB:
    Type: AWS::EC2::Route
    Properties:
      RouteTableId: !Ref PrivateRouteTableA
      DestinationCidrBlock: 10.1.0.0/16
      VpcPeeringConnectionId: !Ref VpcPeeringAB

  PrivateRouteTableB:
    Type: AWS::EC2::RouteTable
    Properties:
      VpcId: !Ref VpcB

  PeeringRouteBtoA:
    Type: AWS::EC2::Route
    Properties:
      RouteTableId: !Ref PrivateRouteTableB
      DestinationCidrBlock: 10.0.0.0/16
      VpcPeeringConnectionId: !Ref VpcPeeringAB

  VpcBPrivateSubnetARouteAssoc:
    Type: AWS::EC2::SubnetRouteTableAssociation
    Properties:
      SubnetId: !Ref VpcBPrivateSubnetA
      RouteTableId: !Ref PrivateRouteTableB

  VpcBPrivateSubnetBRouteAssoc:
    Type: AWS::EC2::SubnetRouteTableAssociation
    Properties:
      SubnetId: !Ref VpcBPrivateSubnetB
      RouteTableId: !Ref PrivateRouteTableB
```

## Expected result

`infra/main.yml` exists with VPC and networking resources. Steps 00049 and 00050 will append further resources and the Outputs section.
