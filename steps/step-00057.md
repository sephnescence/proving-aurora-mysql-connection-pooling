# Step 00057 — Create DEPLOYMENT.md

## Context

A deployment README that documents every manual step required to go from a fresh AWS account to a running environment. This consolidates all one-time prerequisites, ordered by when they must be completed relative to the GitHub Actions workflows.

## Action

Create the file `DEPLOYMENT.md` with the following exact content:

```markdown
# Deployment Guide

This document is an overview of the deployment sequence. Detailed commands for each phase are in the corresponding step files.

## Prerequisites

- AWS CLI configured with sufficient permissions (AdministratorAccess recommended for initial setup)
- `kubectl` installed
- `helm` installed

---

## Deployment sequence

### Phase 1 — Bootstrap and deploy infrastructure (step 51b)

One-time account prerequisites (bootstrap stack, ALB IAM policy, GitHub secret) followed by deploying the main CloudFormation stack, installing Helm charts, and applying Kubernetes manifests. See **step 51b** for full commands.

### Phase 2 — Verify ALB prefix stripping (step 51c)

Confirm the ALB is correctly stripping path prefixes before forwarding to the NestJS apps. See **step 51c** for full commands.

### Phase 3 — Run load tests (steps 54–55)

Trigger the Artillery sequential and concurrent workflows from the GitHub Actions tab.

---

## Teardown

To destroy the environment, trigger `.github/workflows/teardown.yml` manually. To also remove ECR and the OIDC provider, subsequently trigger `.github/workflows/teardown-bootstrap.yml`.
```

## Expected result

`DEPLOYMENT.md` exists at the repo root with a complete pre-deployment checklist.
