# Step 00057 — Create DEPLOYMENT.md

## Context

A deployment README that documents every manual step required to go from a fresh AWS account to a running environment. This consolidates all one-time prerequisites, ordered by when they must be completed relative to the GitHub Actions workflows.

## Action

Create the file `DEPLOYMENT.md` with the following exact content:

```markdown
# Deployment Guide

This document covers every manual step required before and after the GitHub Actions workflows can run. The workflows themselves (`deploy.yml`, `teardown.yml`, etc.) handle everything else automatically.

## Prerequisites

- AWS CLI configured with sufficient permissions (AdministratorAccess recommended for initial setup)
- `kubectl` installed
- `helm` installed
- GitHub repository secrets configured (see below)

---

## 1. Deploy the bootstrap stack (one-time)

The bootstrap stack creates the GitHub OIDC provider, deploy IAM role, and ECR repositories. It must be deployed manually before any workflow can run.

```bash
aws cloudformation deploy \
  --template-file infra/bootstrap.yml \
  --stack-name aurora-pooling-poc-bootstrap \
  --parameter-overrides GitHubOrg=<your-github-org> \
  --capabilities CAPABILITY_NAMED_IAM \
  --region ap-southeast-2
```

---

## 2. Create the ALB Controller IAM policy (one-time)

The `AlbControllerIrsaRole` in `infra/main.yml` references a managed policy named `AWSLoadBalancerControllerIAMPolicy`. This policy must exist in your account before the main stack deploys.

```bash
curl -o alb-iam-policy.json \
  https://raw.githubusercontent.com/kubernetes-sigs/aws-load-balancer-controller/main/docs/install/iam_policy.json

aws iam create-policy \
  --policy-name AWSLoadBalancerControllerIAMPolicy \
  --policy-document file://alb-iam-policy.json \
  --region ap-southeast-2
```

---

## 3. Configure GitHub repository secrets

In your GitHub repository go to **Settings → Secrets and variables → Actions** and add:

| Secret | Value |
|--------|-------|
| `AWS_ACCOUNT_ID` | Your 12-digit AWS account number |

---

## 4. Run the deploy workflow

Trigger `.github/workflows/deploy.yml` manually from the Actions tab on the `main` branch. This will:

1. Build and push Docker images to ECR
2. Deploy `infra/main.yml` (EKS, Aurora, RDS Proxy, IAM, CloudWatch)
3. Update kubeconfig
4. Install Helm charts (ESO, ALB Controller, Fluent Bit)
5. Apply Kubernetes manifests

---

## 5. Verify ALB prefix stripping

See **step 51b** for instructions on verifying and fixing ALB prefix stripping after the deploy workflow completes.

---

## Teardown

To destroy the environment, trigger `.github/workflows/teardown.yml` manually. To also remove ECR and the OIDC provider, subsequently trigger `.github/workflows/teardown-bootstrap.yml`.
```

## Expected result

`DEPLOYMENT.md` exists at the repo root with a complete pre-deployment checklist.
