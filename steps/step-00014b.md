# Step 00014b — Fix pooled-app tsconfig.json

## Context

The NestJS CLI 11 generates `tsconfig.json` with `"module": "nodenext"` and `"moduleResolution": "nodenext"`. NodeNext requires explicit `.js` extensions on all relative imports (e.g. `'./health.controller.js'`), but NestJS code uses bare relative imports. This causes TypeScript errors on every local import.

The fix is to use `"module": "commonjs"`, which is the standard NestJS setup and resolves bare relative imports correctly.

## Action

Replace the entire contents of `apps/pooled-app/tsconfig.json` with:

```json
{
  "compilerOptions": {
    "module": "commonjs",
    "declaration": true,
    "removeComments": true,
    "emitDecoratorMetadata": true,
    "experimentalDecorators": true,
    "allowSyntheticDefaultImports": true,
    "target": "ES2023",
    "sourceMap": true,
    "outDir": "./dist",
    "baseUrl": "./",
    "incremental": true,
    "skipLibCheck": true,
    "strictNullChecks": true,
    "noImplicitAny": true,
    "strictBindCallApply": true,
    "forceConsistentCasingInFileNames": true,
    "noFallthroughCasesInSwitch": true,
    "esModuleInterop": true,
    "isolatedModules": true
  }
}
```

## Expected result

`apps/pooled-app/src` compiles without errors. Bare relative imports like `'./health.controller'` resolve correctly.
